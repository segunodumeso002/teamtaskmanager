# Team Task Manager Backend

AWS Lambda + API Gateway backend scaffold for the Team Task Manager portfolio project.

## Implemented in this scaffold
- Lambda entrypoint and HTTP router
- Endpoint handlers matching `docs/api-contract.md`
- Service layer and repository abstraction
- In-memory repository for local development
- DynamoDB repository implementation for current API endpoints
- Infrastructure templates in `backend/infra` for DynamoDB, IAM, and deploy flow

## Quick start (local)
```bash
cd backend
npm install
npm run start
```

Then set frontend `.env`:
```bash
VITE_API_BASE_URL=http://localhost:4000
```

## Environment
Copy `backend/.env.example` values into your runtime environment.

### Auth modes
- `AUTH_MODE=dev`: accepts `Bearer dev-token-<userId>` generated by current login endpoint.
- `AUTH_MODE=cognito`: verifies JWTs against Cognito JWKS using `COGNITO_USER_POOL_ID` and `COGNITO_CLIENT_ID`.

### Protected routes
- `POST /projects`
- `POST /tasks`
- `PATCH /tasks/{taskId}`
- `DELETE /tasks/{taskId}`
- `POST /tasks/{taskId}/comments`
- `DELETE /comments/{commentId}`

## Deployment note
Use `backend/infra/dynamodb-tables.md`, `backend/infra/iam-policy-lambda-dynamodb.json`, and `backend/infra/deploy-lambda.md` for AWS CLI setup.

For Cognito setup (console click-by-click), use `backend/infra/cognito-setup.md`.
